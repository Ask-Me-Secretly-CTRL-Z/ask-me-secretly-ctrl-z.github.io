Legal & Ethical Foundations · الأسس القانونية والأخلاقية

English: Before writing a single line of code, understand the legal landscape. Always check robots.txt (e.g., example.com/robots.txt) to see which paths are disallowed. Respect rate limits — making too many requests too fast can overload servers and get your IP blocked.

English: Different jurisdictions have different laws. In the US, hiQ Labs v. LinkedIn ruled that scraping publicly accessible data is legal. In the EU, GDPR restricts scraping personal data. In many countries, bypassing technical protections (like login walls) may violate computer fraud laws. Always consult a lawyer if you're unsure.

English: Ethical scraping principles: use public APIs when available, identify your scraper with a descriptive User-Agent, throttle requests to avoid overwhelming servers, never scrape login-protected content without authorization, and don't republish copyrighted data verbatim. Being a good citizen of the web ensures the ecosystem remains open for everyone.

العربية: قبل كتابة أي سطر كود، افهم المشهد القانوني. تحقق دائمًا من robots.txt لترى المسارات الممنوعة. احترم حدود السرعة — الطلبات الكثيرة جدًا يمكن أن تثقل الخوادم وتحظر عنوان IP الخاص بك.

العربية: الولايات القضائية المختلفة لها قوانين مختلفة. في الولايات المتحدة، قضية hiQ Labs v. LinkedIn حكمت أن استخراج البيانات المتاحة للعموم قانوني. في الاتحاد الأوروبي، GDPR يقيد استخراج البيانات الشخصية. في العديد من الدول، تجاوز الحمايات التقنية قد ينتهك قوانين الاحتيال الحاسوبي. استشر دائمًا محاميًا إذا كنت غير متأكد.

العربية: مبادئ الاستخراج الأخلاقي: استخدم APIs العامة عندما تكون متاحة، عرّف أداتك الاستخراجية بـ User-Agent وصفي، خفف الطلبات لتجنب إرهاق الخوادم، لا تستخرج محتوى محميًا بتسجيل الدخول بدون إذن، ولا تعيد نشر بيانات محفوظة الحقوق حرفيًا. كونك مواطنًا صالحًا على الإنترنت يضمن بقاء النظام البيئي مفتوحًا للجميع.

BeautifulSoup — The Foundation · الأساسيات

English: BeautifulSoup is the most popular Python library for parsing HTML and XML. Combined with requests for fetching pages, it handles 80% of scraping tasks. Here's how to extract data from a page:

import requests
from bs4 import BeautifulSoup

url = "https://example.com/articles"
response = requests.get(url, headers={
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
})
soup = BeautifulSoup(response.text, "html.parser")

articles = soup.find_all("article")
for article in articles:
    title = article.find("h2").get_text(strip=True)
    link = article.find("a")["href"]
    summary = article.find(class_="summary").get_text(strip=True)
    print(f"{title}: {link}")

English: This code fetches a page, parses it, then extracts each article's title, link, and summary. The User-Agent header makes the request appear to come from a real browser. BeautifulSoup's find and find_all methods accept tags, attributes, CSS classes, and even custom functions for complex filtering.

English: Advanced BeautifulSoup techniques include navigating the parse tree with .parent, .children, .next_sibling, and using CSS selectors via soup.select(). For broken HTML, BeautifulSoup's parsers (html.parser, lxml, html5lib) handle malformed markup gracefully. Always use lxml for speed — it's 5-10x faster than the default parser.

# Advanced BeautifulSoup techniques

soup.select("div.content > p.summary")
soup.select("#main article[data-type='blog']")

# Navigating the tree
for sibling in soup.find("h2").find_next_siblings("p"):
    print(sibling.text)

# Handling pagination
for page in range(1, 6):
    url = f"https://example.com/page/{page}"
    response = requests.get(url, headers=headers)
    soup = BeautifulSoup(response.text, "lxml")
    # Extract data from each page...

العربية: هذا الكود يجلب الصفحة ويحللها ثم يستخرج عنوان كل مقال ورابطه وملخصه. header User-Agent يجعل الطلب يبدو وكأنه من متصفح حقيقي. دوال find و find_all في BeautifulSoup تقبل الوسوم والخصائص وكلاسات CSS وحتى دوال مخصصة للفلترة المعقدة.

العربية: تقنيات BeautifulSoup المتقدمة تشمل التنقل في شجرة التحليل بـ .parent و .children و .next_sibling واستخدام محددات CSS عبر soup.select(). لـ HTML التالف، محللات BeautifulSoup تتعامل مع الوسوم غير الصحيحة برشاقة. استخدم دائمًا lxml للسرعة — إنه أسرع 5-10 مرات من المحلل الافتراضي كما في الأمثلة أعلاه.

Scrapy — Industrial-Grade Scraping · الاستخراج الصناعي

English: When you need to scrape thousands of pages efficiently, Scrapy is the answer. It's an asynchronous framework that handles concurrency, request scheduling, data pipelines, and export formats out of the box. Create a new Scrapy project with scrapy startproject myproject, define your data model (Item), write a Spider that follows links and extracts data, then run it with scrapy crawl myspider -o output.json.

English: Scrapy's architecture is built for scale. The engine manages a pool of concurrent requests, the scheduler prioritizes URLs, and the downloader handles middlewares for retries, proxies, and user-agent rotation. Items pass through pipelines where you can clean, validate, and store data. Here's a complete spider example:

import scrapy

class BlogSpider(scrapy.Spider):
    name = "blogs"
    start_urls = ["https://example.com/blog"]

    def parse(self, response):
        # Extract articles from listing page
        for article in response.css("article.post"):
            yield {
                "title": article.css("h2::text").get(),
                "url": article.css("a::attr(href)").get(),
                "date": article.css(".date::text").get(),
            }

        # Follow pagination
        next_page = response.css("a.next::attr(href)").get()
        if next_page:
            yield response.follow(next_page, self.parse)

English: Run this with scrapy crawl blogs -o blogs.json. Scrapy automatically handles concurrency, retries failed requests, and respects robots.txt. For large projects, use Item Loaders for data cleaning and Feed Exports for multiple output formats (JSON, CSV, XML, Parquet).

العربية: عندما تحتاج لاستخراج آلاف الصفحات بكفاءة، Scrapy هو الحل. إنه إطار عمل غير متزامن يتولى التزامن وجدولة الطلبات وخطوط أنابيب البيانات وتنسيقات التصدير مباشرة. أنشئ مشروع Scrapy جديد بـ scrapy startproject myproject، حدد نموذج البيانات (Item)، اكتب Spider يتبع الروابط ويستخرج البيانات، ثم شغله بـ scrapy crawl myspider -o output.json.

العربية: معمارية Scrapy مبنية للتوسع. المحرك يدير مجموعة من الطلبات المتزامنة، المجدول يرتب عناوين URL، والتنزيل يتعامل مع البرمجيات الوسيطة لإعادة المحاولة والبروكسيات وتناوب User-Agent. العناصر تمر عبر خطوط أنابيب حيث يمكنك تنظيف البيانات والتحقق منها وتخزينها. المثال أعلاه يوضح Spider كامل يتبع الصفحات ويستخرج المقالات.

Selenium & Playwright — JavaScript Sites · المواقع المعقدة

English: Modern websites heavily rely on JavaScript to render content. BeautifulSoup can't execute JavaScript — it only sees the initial HTML. Selenium and Playwright solve this by controlling a real browser. Playwright is newer, faster, and more reliable:

from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=True)
    page = browser.new_page()
    page.goto("https://example.com")
    page.wait_for_selector(".article-list")
    
    articles = page.query_selector_all(".article")
    for article in articles:
        title = article.query_selector("h2").inner_text()
        print(title)
    
    browser.close()

English: Playwright waits for JavaScript to execute, making it perfect for SPAs, infinite scroll pages, and sites with dynamic content. It also supports network interception, screenshot capture, PDF generation, and mobile viewport emulation. You can intercept API calls made by the page:

English: For infinite scroll pages, Playwright can auto-scroll and wait for new content. Use page.evaluate() to run custom JavaScript in the browser context, perfect for extracting data from JavaScript variables or complex UI states. Playwright is also async-native with playwright.async_api for high-performance concurrent scraping.

# Intercepting network requests with Playwright
def handle_response(response):
    if "/api/data" in response.url:
        data = response.json()
        print(f"Intercepted API data: {len(data)} items")

page.on("response", handle_response)
page.goto("https://example.com")
page.wait_for_timeout(3000)  # Wait for all API calls

العربية: Playwright ينتظر حتى تنفذ JavaScript، مما يجعله مثاليًا لتطبيقات الصفحة الواحدة وصفحات التمرير اللانهائي والمواقع ذات المحتوى الديناميكي. كما أنه يدعم اعتراض الشبكة والتقاط الشاشات وإنشاء PDF ومحاكاة منفذ العرض للجوال. يمكنك اعتراض استدعاءات API التي تقوم بها الصفحة.

العربية: لصفحات التمرير اللانهائي، Playwright يمكنه التمرير تلقائيًا وانتظار المحتوى الجديد. استخدم page.evaluate() لتشغيل JavaScript مخصص في سياق المتصفح، مثالي لاستخراج البيانات من متغيرات JavaScript أو حالات UI المعقدة. Playwright يدعم أيضًا البرمجة غير المتزامنة بـ playwright.async_api للاستخراج المتزامن عالي الأداء كما في المثال أعلاه.

Data Cleaning & Storage · تنظيف البيانات وتخزينها

English: Raw scraped data is almost always messy. Use Pandas for cleaning — remove duplicates, handle missing values, normalize text, and convert data types. Store clean data in CSV, JSON, or directly into databases like SQLite, PostgreSQL, or MongoDB.

English: Here's a complete data cleaning pipeline using Pandas. This handles the most common issues: missing data, inconsistent formatting, duplicates, and type conversion. Always validate your data after cleaning — check for unexpected values, outliers, and data integrity:

import pandas as pd

df = pd.read_json("scraped_data.json")

# Remove duplicates
df = df.drop_duplicates(subset=["url"])

# Handle missing values
df["description"] = df["description"].fillna("No description")
df["price"] = pd.to_numeric(df["price"], errors="coerce")

# Normalize text
df["title"] = df["title"].str.strip().str.title()

# Convert dates
df["date"] = pd.to_datetime(df["date"], errors="coerce")

# Save to database
from sqlalchemy import create_engine
engine = create_engine("postgresql://user:pass@localhost/mydb")
df.to_sql("articles", engine, if_exists="append", index=False)

English: For large datasets, consider Parquet format (compressed, columnar) instead of CSV. For databases, use batch inserts with executemany or ORM bulk operations. Always add indexes on columns you'll query frequently. Set up data validation checks before storage to catch scraping errors early.

العربية: البيانات المستخرجة الخام دائمًا ما تكون غير نظيفة. استخدم Pandas للتنظيف — إزالة التكرارات، معالجة القيم المفقودة، توحيد النص، وتحويل أنواع البيانات. خزّن البيانات النظيفة في CSV أو JSON أو مباشرة في قواعد بيانات مثل SQLite أو PostgreSQL أو MongoDB.

العربية: إليك خط أنابيب تنظيف بيانات كامل باستخدام Pandas كما في المثال أعلاه. هذا يتعامل مع المشكلات الأكثر شيوعًا: البيانات المفقودة، التنسيق غير المتناسق، التكرارات، وتحويل الأنواع. تحقق دائمًا من بياناتك بعد التنظيف — افحص القيم غير المتوقعة والقيم المتطرفة وتكامل البيانات.

Avoiding Anti-Scraping Measures · تجنب إجراءات منع الاستخراج

English: Websites employ various anti-scraping measures. Counter them ethically: rotate User-Agent headers, use proxy rotation (residential proxies are best), add random delays between requests (2-5 seconds), respect robots.txt, and limit request frequency. If a site offers an API, use it instead of scraping.

English: Advanced techniques include using a proxy pool with automatic failover, rotating IP addresses per request, and mimicking human behavior patterns (random mouse movements, natural scrolling). For Cloudflare-protected sites, consider using Playwright with stealth plugins that mask automation signals:

import random
import time
from itertools import cycle

proxies = [
    "http://proxy1:8080",
    "http://proxy2:8080",
    "http://proxy3:8080",
]
proxy_pool = cycle(proxies)

user_agents = [
    "Mozilla/5.0 (Windows NT 10.0; Win64; x64) Firefox/120.0",
    "Mozilla/5.0 (Macintosh; Intel Mac OS X 14_0) Safari/17.0",
    "Mozilla/5.0 (X11; Linux x86_64) Chrome/120.0.0.0",
]

for url in urls:
    proxy = next(proxy_pool)
    headers = {"User-Agent": random.choice(user_agents)}
    
    try:
        response = requests.get(url, proxies={"http": proxy}, headers=headers, timeout=10)
        # Process response...
    except requests.RequestException:
        print(f"Failed: {url}, trying next proxy...")
    
    delay = random.uniform(2, 5)
    time.sleep(delay)

English: Most blocks happen because scrapers are too aggressive. Common mistakes: no delays between requests, same User-Agent for every request, ignoring robots.txt, scraping during peak hours, and not handling HTTP errors gracefully. Build retry logic with exponential backoff — if a request fails, wait 1 second, then 2, then 4, then 8, up to a maximum.

العربية: المواقع تستخدم إجراءات متنوعة لمنع الاستخراج. تصدَّ لها بشكل أخلاقي: غيّر User-Agent headers، استخدم بروكسيات متناوبة، أضف تأخيرات عشوائية (2-5 ثواني)، احترم robots.txt، وحدد تكرار الطلبات. إذا كان الموقع يوفر API، استخدمه بدل الاستخراج.

العربية: التقنيات المتقدمة تشمل استخدام مجموعة بروكسيات مع تجاوز تلقائي للفشل، تدوير عناوين IP لكل طلب، ومحاكاة أنماط السلوك البشري. للمواقع المحمية بـ Cloudflare، استخدم Playwright مع إضافات التخفي التي تخفي إشارات الأتمتة. معظم الحظر يحدث لأن أدوات الاستخراج تكون عدوانية جدًا — ابطئ وكن مهذبًا كما في المثال أعلاه.

Building a Scraping Pipeline · بناء خط أنابيب استخراج

English: Production scrapers need scheduling, monitoring, and alerting. Use Apache Airflow or Celery for scheduling periodic scrapes. Store job metadata (start time, items scraped, errors) in a database. Set up alerts for failures via email, Slack, or SMS.

English: A robust scraping pipeline has several stages: URL discovery and queue management, fetching with retry logic, parsing and extraction, data cleaning and validation, storage, and monitoring. Each stage should be independently testable and fault-tolerant. Use message queues like Redis or RabbitMQ to decouple stages:

# Simple scraping pipeline with error handling
import logging
from typing import List, Dict
from datetime import datetime

class ScrapingPipeline:
    def __init__(self):
        self.logger = logging.getLogger(__name__)
        self.stats = {"started": None, "completed": 0, "failed": 0}
    
    def run(self, urls: List[str]) -> List[Dict]:
        self.stats["started"] = datetime.now()
        results = []
        
        for url in urls:
            try:
                data = self.scrape_url(url)
                cleaned = self.clean_data(data)
                self.store_data(cleaned)
                results.append(cleaned)
                self.stats["completed"] += 1
            except Exception as e:
                self.logger.error(f"Failed to scrape {url}: {e}")
                self.stats["failed"] += 1
        
        self.send_alert_if_needed()
        return results
    
    def scrape_url(self, url): ...
    def clean_data(self, data): ...
    def store_data(self, data): ...
    def send_alert_if_needed(self):
        if self.stats["failed"] > self.stats["completed"] * 0.1:
            self.logger.warning("High failure rate detected!")

English: Monitor your pipeline with metrics: requests per minute, success rate, average response time, and data quality scores. Set up dashboards with Grafana or simply log to a file and use tail for small projects. Always have a kill switch — if a website changes its structure, your pipeline should fail gracefully and alert you.

العربية: أدوات الاستخراج الإنتاجية تحتاج إلى جدولة ومراقبة وتنبيه. استخدم Apache Airflow أو Celery لجدولة عمليات الاستخراج الدورية. خزّن بيانات التعريف (وقت البدء، العناصر المستخرجة، الأخطاء) في قاعدة بيانات. أعد تنبيهات للفشل عبر البريد الإلكتروني أو Slack أو SMS.

العربية: خط أنابيب استخراج قوي له عدة مراحل: اكتشاف URL وإدارة قائمة الانتظار، الجلب مع منطق إعادة المحاولة، التحليل والاستخراج، تنظيف البيانات والتحقق منها، التخزين، والمراقبة. كل مرحلة يجب أن تكون قابلة للاختبار بشكل مستقل ومتسامحة مع الأخطاء. استخدم قوائم انتظار الرسائل مثل Redis أو RabbitMQ لفصل المراحل كما في المثال أعلاه.

Real-World Scraping Project · مشروع استخراج عملي

English: Let's combine everything into a real project: scraping an e-commerce site for price monitoring. The pipeline should discover product pages, extract prices and descriptions, handle pagination, avoid blocks, clean the data, and store it in a database for trend analysis.

English: Key considerations for e-commerce scraping: products change frequently, websites have complex structures, session handling may be needed, and competitors may actively block scrapers. Build in features like product change detection, price history tracking, and automatic re-scraping of stale products:

# E-commerce price monitor skeleton
class EcommerceMonitor:
    def __init__(self, db_url: str):
        self.engine = create_engine(db_url)
    
    def scrape_product(self, url: str) -> Dict:
        response = requests.get(url, headers=random_headers())
        soup = BeautifulSoup(response.text, "lxml")
        
        return {
            "url": url,
            "name": soup.select_one("h1.product-name").text.strip(),
            "price": self.extract_price(soup),
            "currency": "USD",
            "in_stock": bool(soup.select_one(".add-to-cart")),
            "timestamp": datetime.now(),
        }
    
    def detect_price_change(self, product: Dict) -> bool:
        last = pd.read_sql(
            "SELECT price FROM products WHERE url = %s ORDER BY timestamp DESC LIMIT 1",
            self.engine, params=[product["url"]]
        )
        return last.empty or last.iloc[0]["price"] != product["price"]

English: Run this daily with a cron job or Airflow DAG. Log all price changes and send alerts when products drop below target prices. With historical data, you can predict price trends and find the best times to buy.

العربية: دعنا نجمع كل شيء في مشروع حقيقي: استخراج موقع تجارة إلكترونية لمراقبة الأسعار. خط الأنابيب يجب أن يكتشف صفحات المنتجات ويستخرج الأسعار والوصف ويتعامل مع التقسيم ويتجنب الحظر وينظف البيانات ويخزّنها في قاعدة بيانات لتحليل الاتجاهات.

العربية: اعتبارات رئيسية لاستخراج التجارة الإلكترونية: المنتجات تتغير باستمرار، المواقع لها هياكل معقدة، قد تحتاج للتعامل مع الجلسات، والمنافسون قد يمنعون أدوات الاستخراج بنشاط. ابنِ ميزات مثل كشف تغيير المنتج وتتبع تاريخ الأسعار وإعادة الاستخراج التلقائي للمنتجات القديمة كما في المثال أعلاه.

Build something useful, respect the websites you scrape from, and contribute back to the open-source tools that make this possible.

العربية: استخراج بيانات الويب بايثون هو قدرة خارقة. ابدأ بـ BeautifulSoup للمشاريع البسيطة، ثم انتقل لـ Scrapy للتوسع، واستخدم Playwright للمواقع المعقدة. استخرج دائمًا بشكل أخلاقي، احترم قواعد المواقع، واستخدم البيانات بمسؤولية.

العربية: تذكر المبادئ الأساسية: افحص robots.txt، استخدم تأخيرات مهذبة، غيّر البروكسيات والـ headers، نظّف بياناتك، راقب خطوط الأنابيب الخاصة بك، والتزم بالقانون. الويب هو أكبر مجموعة بيانات تم إنشاؤها على الإطلاق — وبايثون هو مفتاحك لفتحها. ابنِ شيئًا مفيدًا، احترم المواقع التي تستخرج منها، وساهم في الأدوات مفتوحة المصدر التي تجعل هذا ممكنًا.
