Git Mastery · إتقان الجيت

Introduction · مقدمة

English: Git is the backbone of modern software development. Whether you work solo or in a team of 100, mastering Git will save you hours of frustration. Created by Linus Torvalds in 2005, Git is now used by over 90% of developers worldwide. This guide covers everything from daily commands to advanced workflows that professional teams use.

العربية: Git هو العمود الفقري لتطوير البرمجيات الحديث. سواء كنت تعمل منفردًا أو في فريق من 100، إتقان الجيت سيوفر لك ساعات من الإحباط. أنشأه لينوس تورفالدس في 2005، ويستخدمه الآن أكثر من 90% من المطورين عالميًا. هذا الدليل يغطي كل شيء من الأوامر اليومية إلى سير العمل المتقدم الذي تستخدمه الفرق المحترفة.

Essential Commands · الأوامر الأساسية

English: These commands will cover 90% of your daily Git usage. Master them before moving to advanced topics. Each command has a specific purpose and understanding when to use each one is key to Git proficiency.

العربية: هذه الأوامر ستغطي 90% من استخدامك اليومي لـ Git. أتقنها قبل الانتقال للمواضيع المتقدمة. كل أمر له غرض محدد وفهم متى تستخدم كل منها هو مفتاح إتقان Git.

# Starting a project
git init                    # Initialize a new repository
git clone <url>             # Copy an existing repository

# Daily workflow
git status                  # Check what's changed
git add <file>              # Stage specific file
git add .                   # Stage all changes
git commit -m "message"     # Save staged changes
git log --oneline           # View commit history
git diff                    # See unstaged changes
git diff --staged           # See staged changes

# Branching
git branch                  # List branches
git branch <name>           # Create a new branch
git checkout <branch>       # Switch to a branch
git checkout -b <name>      # Create and switch to new branch
git merge <branch>          # Merge a branch into current

# Remote operations
git push origin <branch>    # Push commits to remote
git pull origin <branch>    # Pull latest from remote
git fetch                   # Download remote data without merging
git remote -v               # List remote repositories

Branching Strategies · استراتيجيات الفروع

English: Choosing the right branching strategy is critical for team productivity. Here are the three most popular approaches with their pros and cons, so you can choose what fits your team's workflow.

العربية: اختيار استراتيجية الفروع المناسبة أمر حاسم لإنتاجية الفريق. إليك الطرق الثلاث الأكثر شيوعًا مع إيجابياتها وسلبياتها، لتختار ما يناسب سير عمل فريقك.

English: Git Flow uses main, develop, feature/, release/, and hotfix/ branches. It's comprehensive but complex — best for large teams with scheduled releases. GitHub Flow keeps it simpler — main and feature branches with PR reviews. Everything in main is deployable. GitLab Flow adds environment branches (staging, production) for more control. For most teams, GitHub Flow is the sweet spot: simple enough to follow, powerful enough for production.

العربية: Git Flow يستخدم فروع main و develop و feature/ و release/ و hotfix/. شامل لكنه معقد — الأفضل للفرق الكبيرة مع إصدارات مجدولة. GitHub Flow أبسط — main وفروع feature مع مراجعات PR. كل شيء في main قابل للنشر. GitLab Flow يضيف فروع بيئة (staging، production) لمزيد من التحكم. لمعظم الفرق، GitHub Flow هو النقطة المثلى: بسيط بما يكفي للمتابعة، قوي بما يكفي للإنتاج.

# GitHub Flow example
git checkout -b feature/user-auth
# ... make changes ...
git add .
git commit -m "Add user authentication"
git push origin feature/user-auth
# Create PR on GitHub, get reviews, merge to main
git checkout main
git pull origin main
git branch -d feature/user-auth  # Clean up local branch

Merge Conflicts · صراعات الدمج

English: Conflicts happen when two people edit the same line in a file. Git can't automatically decide which version to keep. Don't panic — conflicts are normal and resolvable. Use git mergetool or resolve manually in your editor. The conflict markers look like this:

العربية: تحدث صراعات عندما يعدل شخصان نفس السطر في ملف. Git لا يمكنه تحديد أي نسخة يحتفظ بها تلقائيًا. لا داعي للذعر — الصراعات طبيعية وقابلة للحل. استخدم git mergetool أو حلّها يدويًا في المحرر. علامات الصراع تبدو هكذا:

# Conflict markers in a file
<<<<<<< HEAD
console.log("This is my version");
=======
console.log("This is their version");
>>>>>>> feature-branch

# To resolve: edit the file, remove markers, keep the correct version
console.log("This is the resolved version");

# After resolving:
git add resolved-file.js
git commit -m "Resolve merge conflict in resolved-file.js"

English: Best practices to avoid conflicts: communicate with your team about who's working on what, pull frequently (git pull --rebase), keep feature branches short-lived (merge within 1-2 days), break large features into smaller, independent tasks, and use code owners to assign responsibility for specific files.

العربية: أفضل الممارسات لتجنب الصراعات: تواصل مع فريقك حول من يعمل على ماذا، اسحب التحديثات باستمرار (git pull --rebase)، ابقِ فروع الميزات قصيرة العمر (ادمج خلال 1-2 يوم)، قسم الميزات الكبيرة إلى مهام أصغر مستقلة، واستخدم مالكي الكود لتعيين مسؤولية ملفات محددة.

Undoing Changes · التراجع عن التغييرات

English: Everyone makes mistakes. Git provides several escape hatches depending on how far back you need to go. Understanding the difference between these commands prevents accidental data loss.

العربية: الجميع يخطئ. Git يوفر عدة مخارج للطوارئ حسب المدى الذي تحتاج للعودة إليه. فهم الفرق بين هذه الأوامر يمنع فقدان البيانات العرضي.

# Discard unstaged changes
git restore <file>
git checkout -- <file>       # Older syntax

# Unstage a file (keep changes)
git restore --staged <file>
git reset HEAD <file>        # Older syntax

# Amend the last commit (fix message or add forgotten files)
git commit --amend -m "Fixed message"

# Reset to a previous commit (careful!)
git reset --soft HEAD~1      # Keep changes staged
git reset --mixed HEAD~1     # Keep changes unstaged (default)
git reset --hard HEAD~1      # Delete changes completely (dangerous!)

# Revert a commit (safe for shared branches)
git revert <commit-hash>     # Creates a new commit that undoes changes

Advanced Tips · نصائح متقدمة

English: These commands separate beginners from experts. Use them to maintain a clean history, debug efficiently, and work smarter. Practice these in a throwaway repository before using them on important work.

العربية: هذه الأوامر تفصل المبتدئين عن الخبراء. استخدمها للحفاظ على تاريخ نظيف وتصحيح الأخطاء بكفاءة والعمل بذكاء. تدرب عليها في مستودع تجريبي قبل استخدامها في عمل مهم.

# Interactive rebase - clean up commit history
git rebase -i HEAD~5   # Squash, reorder, edit last 5 commits

# Cherry-pick - apply specific commits to current branch
git cherry-pick <commit-hash>

# Stash - save unfinished work temporarily
git stash                # Save changes
git stash list           # View stashes
git stash pop            # Apply and remove latest stash
git stash apply stash@{2} # Apply specific stash without removing

# Bisect - find the commit that introduced a bug
git bisect start
git bisect bad           # Current commit is broken
git bisect good <hash>   # This commit was working
# Git will checkout commits for you to test
git bisect good          # or: git bisect bad
git bisect reset         # When done

# Git log visualizations
git log --graph --oneline --all --decorate

# Find who changed what in a file
git blame <file>

English: Interactive rebase (git rebase -i) is your most powerful tool for maintaining a clean git history. Use it to squash "WIP" and "fix" commits into meaningful, well-described commits before pushing to shared branches. A clean history makes code reviews easier and debugging faster.

العربية: rebase التفاعلي (git rebase -i) هو أقوى أداة لك للحفاظ على تاريخ Git نظيف. استخدمه لدمج commits التجريبية وإصلاحات الأخطاء في commits ذات معنى ووصف جيد قبل الدفع إلى الفروع المشتركة. التاريخ النظيف يجعل مراجعة الكود أسهل وتصحيح الأخطاء أسرع.

GitHub Collaboration · التعاون على GitHub

English: Modern development is collaborative. Learn the GitHub flow: fork the repository, create a feature branch, make changes, push, and open a Pull Request. Write descriptive PR titles and templates. Request reviews from teammates and address feedback. Use GitHub Issues for tracking work and Projects for sprint planning. Protect your main branch with branch protection rules — require PR reviews, status checks, and up-to-date branches before merging.

العربية: التطوير الحديث تعاوني. تعلم تدفق GitHub: انسخ المستودع (fork)، أنشئ فرع ميزة، أجرِ تغييرات، ادفع، وافتح طلب سحب (PR). اكتب عناوين وصفية لطلبات السحب. اطلب مراجعات من زملائك وتعامل مع الملاحظات. استخدم Issues لتتبع العمل و Projects لتخطيط السباقات. احمِ فرعك الرئيسي بقواعد حماية الفروع — تطلب مراجعات PR وفحوصات الحالة.

Git Config & Aliases · إعدادات Git والاختصارات

English: Customize Git with aliases to speed up your workflow. These aliases turn long commands into short ones — instead of typing git log --graph --oneline --all --decorate, just type git lg. Set up your identity, default editor, and line ending preferences once and forget about them:

العربية: خصص Git باختصارات لتسريع سير عملك. هذه الاختصارات تحول الأوامر الطويلة إلى قصيرة — بدلاً من كتابة git log --graph --oneline --all --decorate، اكتب فقط git lg. هيّئ هويتك ومحررك الافتراضي وتفضيلات نهايات الأسطر مرة واحدة:

# Basic configuration
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global core.editor "code --wait"
git config --global init.defaultBranch main

# Useful aliases
git config --global alias.lg "log --graph --oneline --all --decorate"
git config --global alias.s "status -s"
git config --global alias.cm "commit -m"
git config --global alias.co "checkout"
git config --global alias.br "branch"
git config --global alias.df "diff"
git config --global alias.dfs "diff --staged"
git config --global alias.unstage "restore --staged"
git config --global alias.last "log -1 HEAD"
git config --global alias.undo "reset --soft HEAD~1"

# After this, you can type:
git s          # instead of git status -s
git lg         # instead of git log --graph --oneline --all --decorate
git cm "fix"   # instead of git commit -m "fix"
git undo       # undo last commit (keep changes)

English: These aliases alone can save you hundreds of keystrokes per day. Share your aliases with your team by committing a .gitconfig template to your repository. You can also configure Git hooks — scripts that run automatically on certain events. A pre-commit hook can run linting and tests before allowing a commit, preventing bad code from entering the repository.

العربية: هذه الاختصارات وحدها يمكنها توفير مئات الضغطات يوميًا. شارك اختصاراتك مع فريقك بحفظ قالب .gitconfig في مستودعك. يمكنك أيضًا تكوين hooks في Git — نصوص برمجية تعمل تلقائيًا عند أحداث معينة. pre-commit hook يمكنه تشغيل فحص الكود والاختبارات قبل السماح بالـ commit، مما يمنع الكود السيئ من دخول المستودع.

Conclusion · الخلاصة

English: Git mastery comes with practice. Start with the basic add-commit-push cycle, then gradually incorporate branching, merging, and rebasing into your workflow. Use git rebase -i to clean up history, git cherry-pick to apply specific commits, git stash to save unfinished work, and git bisect to find the commit that introduced a bug. The best Git users are the ones who make the most mistakes — because they learn what each command actually does. Don't be afraid to experiment in a safe branch.

العربية: إتقان Git يأتي مع الممارسة. ابدأ بدورة add-commit-push الأساسية، ثم أدرج تدريجيًا الفروع والدمج و rebase في سير عملك. أفضل مستخدمي Git هم أولئك الذين يرتكبون معظم الأخطاء — لأنهم يتعلمون ماذا يفعل كل أمر فعليًا. لا تخف من التجربة في فرع آمن.
