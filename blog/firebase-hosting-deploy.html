Firebase Hosting · انشر موقعك مجانًا

Why Firebase Hosting? · لماذا فايربيز هوستينج؟

English: Firebase Hosting offers fast, secure, and free static hosting with global CDN, automatic SSL, custom domains, and one-command deployments via the Firebase CLI. It's optimized for modern web apps — single-page apps (React, Vue, Angular), static sites, and progressive web apps. With Google's global edge network, your site loads fast anywhere in the world.

العربية: Firebase Hosting يقدم استضافة سريعة وآمنة ومجانية مع CDN عالمي و SSL تلقائي ونطاقات مخصصة ونشر بأمر واحد عبر Firebase CLI. إنه محسّن لتطبيقات الويب الحديثة — تطبيقات الصفحة الواحدة (React، Vue، Angular) والمواقع الثابتة وتطبيقات الويب التدريجية. مع شبكة Google العالمية، موقعك يُحمل بسرعة في أي مكان في العالم.

Step 1: Prerequisites · المتطلبات الأساسية

English: Before you start, make sure you have: Node.js v18+ installed (node --version to check), a Google account (Gmail), a Firebase project created in the Firebase Console (console.firebase.google.com), and your website files ready (a build folder with index.html at minimum). If you're using React, Next.js, or Vue, build your project first with npm run build.

العربية: قبل أن تبدأ، تأكد من أن لديك: Node.js v18+ مثبتًا (تحقق بـ node --version)، حساب Google (Gmail)، مشروع Firebase منشأ في Firebase Console، وملفات موقعك جاهزة (مجلد build مع index.html على الأقل). إذا كنت تستخدم React أو Next.js أو Vue، ابنِ مشروعك أولاً بـ npm run build.

Step 2: Install & Login · التثبيت وتسجيل الدخول

English: Install the Firebase CLI globally with npm. Then authenticate with your Google account. The login opens a browser window — complete the authentication there. Once logged in, verify with firebase list to see your projects.

العربية: ثبّت Firebase CLI عالميًا باستخدام npm. ثم سجل الدخول بحساب Google. تسجيل الدخول يفتح نافذة متصفح — أكمل المصادقة هناك. بعد تسجل الدخول، تحقق بـ firebase list لرؤية مشاريعك.

# Install Firebase CLI
npm install -g firebase-tools

# Verify installation
firebase --version

# Login to Google
firebase login

# List your Firebase projects
firebase list

Step 3: Initialize · تهيئة المشروع

English: Run firebase init hosting in your project directory. The CLI will ask you to select an existing Firebase project or create a new one. Specify your public directory (usually build, dist, or public). Answer "Yes" when asked if this is a single-page app (this configures URL rewrites for client-side routing). The init command creates two files: firebase.json (configuration) and .firebaserc (project alias).

العربية: شغّل firebase init hosting في مجلد مشروعك. CLI سيطلب منك تحديد مشروع Firebase موجود أو إنشاء مشروع جديد. حدد مجلد العام الخاص بك (عادة build أو dist أو public). أجب بـ "نعم" عندما يُسأل إذا كان هذا تطبيق صفحة واحدة (هذا يهيئ إعادة كتابة URL للتوجيه من جانب العميل). أمر init ينشئ ملفين: firebase.json (الإعدادات) و .firebaserc (اسم المشروع).

# Initialize hosting
firebase init hosting

# You'll be prompted for:
# ? Select a Firebase project: (select from list)
# ? What do you want to use as your public directory? (build)
# ? Configure as a single-page app? (Yes)
# ? Set up automatic builds with GitHub? (No for now)
# ? File build/index.html already exists. Overwrite? (No)

Step 4: Deploy · النشر

English: Run firebase deploy to push your site live in seconds. Your site is served at your-project.web.app and your-project.firebaseapp.com. For subsequent deployments, the same command updates your site with zero downtime. Use firebase deploy --only hosting to deploy only the hosting part if you have other Firebase services.

العربية: شغّل firebase deploy لنشر موقعك في ثوانٍ. موقعك يكون متاحًا على your-project.web.app و your-project.firebaseapp.com. للنشرات اللاحقة، نفس الأمر يُحدّث موقعك بدون توقف. استخدم firebase deploy --only hosting لنشر جزء الاستضافة فقط إذا كان لديك خدمات Firebase أخرى.

# Deploy to production
firebase deploy

# Deploy only hosting (faster)
firebase deploy --only hosting

# Preview deployment (local server)
firebase serve

# Deploy to a specific project
firebase deploy --project my-project-name

English: For team workflows, set up automatic deployments with GitHub Actions. Create a workflow file that runs tests, builds your app, and deploys to Firebase on every push to the main branch. Use Firebase CI token (firebase login:ci) for non-interactive authentication in CI pipelines.

العربية: لسير عمل الفريق، أعد نشرًا تلقائيًا مع GitHub Actions. أنشئ ملف workflow يشغل الاختبارات ويبني تطبيقك وينشر إلى Firebase مع كل push إلى الفرع الرئيسي. استخدم رمز Firebase CI (firebase login:ci) للمصادقة غير التفاعلية في خطوط CI.

Step 5: Custom Domain · نطاق مخصص

English: In Firebase Console, go to Hosting, click "Add custom domain". Enter your domain (e.g., example.com). Firebase will provide DNS records to add at your domain registrar. For the apex domain (example.com), add A records pointing to Firebase's IPs. For www subdomain (www.example.com), add a CNAME record pointing to your-project.web.app. SSL is provisioned automatically by Google — typically within minutes.

العربية: في Firebase Console، اذهب إلى Hosting، انقر "Add custom domain". أدخل نطاقك (مثل example.com). Firebase سيوفر سجلات DNS لتضيفها عند مسجل النطاق. للنطاق الجذر (example.com)، أضف سجلات A تشير إلى عناوين IP الخاصة بـ Firebase. للنطاق الفرعي www (www.example.com)، أضف سجل CNAME يشير إلى your-project.web.app. SSL يتم توفيره تلقائيًا بواسطة Google — عادة في دقائق.

# Typical DNS records for Firebase Hosting
# For apex domain (example.com):
#   Type: A     Name: @      Value: 151.101.1.91
#   Type: A     Name: @      Value: 151.101.65.91
#   Type: A     Name: @      Value: 151.101.129.91
#   Type: A     Name: @      Value: 151.101.193.91
#
# For www subdomain:
#   Type: CNAME Name: www    Value: your-project.web.app.

Firebase.json Configuration · إعدادات Firebase.json

English: The firebase.json file controls hosting behavior. Configure rewrites for SPA routing, redirects for SEO, headers for security, and clean URLs. Here's a production-ready configuration:

العربية: ملف firebase.json يتحكم في سلوك الاستضافة. هيّئ إعادة الكتابة لتوجيه SPA وإعادة التوجيه لتحسين محركات البحث والروؤس للأمان وعناوين URL النظيفة. إليك إعداد جاهز للإنتاج:

{
  "hosting": {
    "public": "build",
    "ignore": ["firebase.json", "**/.*", "**/node_modules/**"],
    "rewrites": [
      { "source": "**", "destination": "/index.html" }
    ],
    "redirects": [
      { "source": "/old-page", "destination": "/new-page", "type": 301 },
      { "source": "/blog/:slug", "destination": "/articles/:slug", "type": 301 }
    ],
    "headers": [
      {
        "source": "**/*.@(js|css|jpg|png|svg|webp)",
        "headers": [
          { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
        ]
      },
      {
        "source": "**",
        "headers": [
          { "key": "X-Frame-Options", "value": "DENY" },
          { "key": "X-Content-Type-Options", "value": "nosniff" },
          { "key": "Referrer-Policy", "value": "strict-origin-when-cross-origin" }
        ]
      }
    ],
    "appAssociation": "AUTO"
  }
}

English: The rewrites section sends all requests to index.html — essential for SPAs with client-side routing. The redirects section handles 301 permanent redirects for SEO. The headers section sets aggressive caching for static assets and security headers for all pages. Add a 404.html page for custom 404 handling.

العربية: قسم rewrites يرسل كل الطلبات إلى index.html — ضروري لتطبيقات الصفحة الواحدة. قسم redirects يتعامل مع إعادة التوجيه الدائم 301 لتحسين محركات البحث. قسم headers يضبط تخزينًا مؤقتًا قويًا للأصول الثابتة وروؤس أمان لجميع الصفحات. أضف صفحة 404.html للتعامل المخصص مع خطأ 404.

CI/CD with GitHub Actions · النشر التلقائي

English: Automate deployments so every push to main deploys your site automatically. Generate a Firebase CI token with firebase login:ci, add it as a repository secret named FIREBASE_TOKEN, then create this workflow:

العربية: أتمتة النشر بحيث كل push إلى main ينشر موقعك تلقائيًا. أنشئ رمز Firebase CI بـ firebase login:ci، أضفه كسر للمستودع باسم FIREBASE_TOKEN، ثم أنشئ هذا workflow:

# .github/workflows/deploy.yml
name: Deploy to Firebase
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run build
      - uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: "${{ secrets.GITHUB_TOKEN }}"
          firebaseServiceAccount: "${{ secrets.FIREBASE_SERVICE_ACCOUNT }}"
          channelId: live
          projectId: your-project-id

Preview Channels · قنوات المعاينة

English: Firebase Hosting also supports preview channels — each Pull Request gets its own unique URL. This is a game-changer for team workflows. When you open a PR, Firebase automatically deploys a preview version of your site with a unique URL (e.g., pr-42-my-project.web.app). You (and your reviewers) can see the changes live before merging. To configure this, add the --channel-id flag to your deploy command or use the channelId: "preview" option in GitHub Actions. Preview channels expire after 7 days by default, keeping your deployment dashboard clean.

العربية: Firebase Hosting يدعم أيضًا قنوات المعاينة — كل طلب سحب يحصل على URL فريد خاص به. هذا تغيير جذري لسير عمل الفريق. عندما تفتح PR، Firebase ينشر تلقائيًا نسخة معاينة من موقعك مع URL فريد (مثل pr-42-my-project.web.app). يمكنك أنت والمراجعون رؤية التغييرات مباشرة قبل الدمج. لتكوين هذا، أضف علامة --channel-id لأمر النشر أو استخدم خيار channelId: "preview" في GitHub Actions. قنوات المعاينة تنتهي بعد 7 أيام افتراضيًا، مما يحافظ على نظافة لوحة النشر الخاصة بك.

Troubleshooting · استكشاف الأخطاء

English: Common issues: "Hosting has no files" — make sure your public directory is correct and you've built your app. "SSL not provisioned" — custom domain SSL can take up to 24 hours. "Deploy fails" — check that you're logged in (firebase login) and have the correct project selected. "404 on page refresh for SPAs" — ensure your firebase.json has the rewrite rule { "source": "**", "destination": "/index.html" }. "CORS errors on API calls" — configure CORS on Cloud Functions or use a proxy rewrite in firebase.json.

العربية: مشاكل شائعة: "Hosting has no files" — تأكد أن مجلد العام صحيح وأنك بنيت تطبيقك. "SSL not provisioned" — SSL للنطاق المخصص قد يستغرق حتى 24 ساعة. "Deploy fails" — تحقق أنك مسجل الدخول (firebase login) والمشروع الصحيح محدد. "404 عند تحديث الصفحة لتطبيقات SPA" — تأكد أن firebase.json لديه قاعدة إعادة الكتابة { "source": "**", "destination": "/index.html" }. "أخطاء CORS في استدعاءات API" — هيّئ CORS على Cloud Functions أو استخدم إعادة كتابة وكيل في firebase.json.

Conclusion · الخلاصة

English: Firebase Hosting is one of the easiest and most reliable ways to deploy web applications. Free SSL, global CDN, one-command deployment, and custom domain support make it ideal for both beginners and professionals. Combine it with GitHub Actions for automatic deployments, and you have a world-class hosting setup at zero cost. Start deploying today — your first site can be live in under 5 minutes.

العربية: Firebase Hosting هي واحدة من أسهل وأكثر الطرق موثوقية لنشر تطبيقات الويب. SSL مجاني و CDN عالمي ونشر بأمر واحد ودعم النطاقات المخصصة تجعله مثاليًا للمبتدئين والمحترفين. ادمجه مع GitHub Actions للنشر التلقائي وستحصل على إعداد استضافة عالمي المستوى بدون تكلفة. ابدأ النشر اليوم — أول موقع لك يمكن أن يكون متاحًا في أقل من 5 دقائق.
